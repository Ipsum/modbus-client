Troubleshooting

Please follow these general troubleshooting steps first:

  • Ensure the clark Sonic device is powered.
  • Ensure a jumper has been inserted across the Commissioning pins of the device.
  • Ensure the clark Sonic device is connected to the computer with the Commissioning software installed.
  • Ensure you have the correct COM port selected in the Commissioning software.

Compatible Operating Systems

  • Microsoft Windows: XP, Vista, 7
  • Mac OS X

Error Codes

The serial port could not be opened

The likely cause of this is an unpowered or disconnected device. Having the wrong COM port selected in the clark Sonic Commissioning software could also cause this. Finally, another software program trying to use the same COM port will also cause this error to display.

A serial communications error has occurred

Recheck the general troubleshooting steps and try again.

There was no response from the meter, please check all connections

The device is either powered off, on a different COM port, or busy. A COM port connection was successfully opened, but the meter did not acknowledge your message.

The response from the meter was incorrect. Please check that the jumper correctly set.

This could be caused by interference or noise in the communication line, a problem with the MODBUS network, the wrong device id being selected if on a network with other devices, or the device being busy.

Error Reading Config File

If this error persists, your installation of the clark Sonic Commissioning software has been corrupted. Please uninstall, redownload, and reinstall. This could also be caused by a non-compatible operating system. See the top of this file for a list of compatible operating systems.

Could not create log file

The directory in which you tried to save the log file might be write protected (read only) or the file name might be invalid.
